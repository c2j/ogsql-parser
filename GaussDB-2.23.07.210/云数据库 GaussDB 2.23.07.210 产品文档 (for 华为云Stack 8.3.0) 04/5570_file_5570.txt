节点修复 节点修复 故障现象 数据库集群是由多台主机组成的，当集群中主机发生故障时，为了使 GaussDB 快速地恢复正常，运维人员可以使用节点修复功能修复发生故障的节点。 以混合部署（4副本）为例，集群部署在多个AZ(数据中心)，如3AZ部署、同城双活等多AZ部署场景，节点上的进程损坏且无法修复，如下所示。 图1 分布式集群混合部署示意 图2 主备版集群示意图 业务影响 如果主DN/GTM/CMS组件故障，将会触发主备切换，业务有10S的闪断；如果某个备DN/GTM/CMS故障，则对业务没有影响，但集群可靠性会降低。CN发生故障时，故障CN在被自动剔除前集群不能做DDL操作。 注意事项 修复操作需要确保集群处于启动状态，且没有加锁。 如果集群中部署有ETCD，则正常的ETCD个数必须大于ETCD总个数的一半，故障的ETCD才可以被修复。 实例磁盘满状态下不允许进行节点修复。 如果故障节点所在主机的安装目录下（$GAUSSHOME/bin/）的二进制文件损坏或丢失，则不能通过该章节进行修复。需要复制其他正常主机对应的二进制文件到该主机，再通过该章节修复。 主DN存活时，故障的备DN才可以被修复。 主GTM存活时，故障的备GTM才可以被修复（仅分布式集群混合部署）。 在前一次修复结束后才能再次执行修复，禁止多个节点同时执行修复。 ETCD异常时，修复需要等待一段时间才可执行。 预期处理时长 与故障组件的数据量有关。 日志收集要求 参考 下载租户节点日志 收集各模块日志路径。 Agent： /home/Ruby/log/asyncJobResult.dat /home/Ruby/log/agent.log GaussDB Kernel： /var/lib/log/Ruby 处理思路 在DBS运维管理平台上通过节点修复功能，修复故障的进程。 故障定位 某节点的CN/DN/GTM/ETCD/CMS发生故障，或整个节点故障，收到故障告警。 在DBS运维管理平台实例管理界面的实例列表中单击实例，在实例基本信息页查看节点状态。 图3 节点状态 故障恢复 登录DBS运维管理平台 。 在“实例运维 > 实例管理”页签根据实例ID找到故障节点所在实例。单击实例名称。 找到故障节点，在对应节点的“操作”列，单击 “更多 >节点修复” 。 图4 节点修复 ETCD组件故障后，组件所在节点不会异常，需要查看ETCD组件是否异常。 在弹出框中选择“节点修复”，单击“确定”。 图5 节点修复确认 确认集群均衡状态。 参考 登录实例节点 ，登录数据库节点。 执行如下命令，导入环境变量。 source /home/Ruby/gauss_env_file 执行如下指令，查询集群状态。 c m_ctl query -Civd 如图所示，balanced参数为“No”，说明集群主备分布不均衡。需要继续执行下列步骤，使主备分布均衡，如果主备分布已经均衡，则不需要执行如下步骤。 执行如下指令，使集群主备分布均衡。 cm_ctl switchover -a 该操作建议在业务压力较小时执行。 执行如下指令，查询主备分布是否均衡。 cm_ctl query -Civd 如图所示，balanced参数为“Yes”，说明集群主备分布均衡。 恢复确认 通过在DBS运维管理平台查看节点状态是否恢复正常。 图6 恢复确认 父主题： 租户面应急案例 版权所有 © 华为技术有限公司
