NAS服务器共享目录配置 NAS服务器共享目录配置 数据库备份到NAS服务器，需要NAS服务器挂载目录能被数据库实例的Ruby用户正常访问。配置方式可以参考下面的步骤，以共享目录/home/nfs为例： 以root用户登录NAS服务器。 查看rpcbind状态 systemctl status rpcbind systemctl status nfs 如果该服务为停止状态，需要先启用rpcbind 启动rpcbind： systemctl start rpcbind 再启动nfs： systemctl start nfs 执行如下命令，查看NFS共享主配置文件。 vim /etc/exports 查看是否有/home/nfs目录的共享配置。如果没有，在该文件末尾添加如下一行。 /home/nfs *(insecure,rw,sync,no_subtree_check,all_squash) 其中all_squash表示客户端访问共享目录时会被映射成匿名用户。 执行如下命令，进入挂载目录。 cd /home /home请根据实际挂载点替换。 创建挂载点。 mkdir nfs 当前操作需要是root账户。如已创建挂载点，可忽略该步骤。 执行如下命令，切换nfs的所属用户和权限。 chown nobody:nobody nfs 执行如下命令，授予挂载点目录读写权限。 chmod 755 nfs 执行如下命令，重新共享目录。 exportfs -rv 父主题： NAS配置最佳实践 版权所有 © 华为技术有限公司
