管理面元数据库同步异常 管理面元数据库同步异常 故障现象 管理面元数据库主备同步异常。 故障影响 管理面元数据库数据同步异常，如果数据库发生主备倒换，会导致DBS管理面部分数据丢失，进而会导致数据库服务功能使用，影响实例查询、备份等与管理面元数据库有交互的操作。 预期恢复时长 30分钟。 处理思路 登录数据库节点，通过检查脚本检查同步状态，通过修复脚本恢复。 故障定位 数据库主备节点延迟较大。 数据库单节点故障恢复后主备不同步。 故障恢复 参考 登录数据库管理节点 ，登录到管理面元数据库节点。 切换到root用户。 sudo su - root 默认密码请参考《华为云Stack 8.3.0 帐户一览表》中 “数据库” 页签。根据“云服务”、“帐户所在的管理虚拟机/节点”列，并且帐户名称选择 “root” 获取。 切换dbadmin用户。 su dbadmin 执行如下命令，查看节点角色。 gs_ctl query 如果LOCAL_ROLE值为Primary，则为master节点，若为Standby则是slave节点。 执行以下命令，查看数据同步状态。 python /opt/gaussdb/dbmha/haprogram/db_tool.py db_status 若“ha_status”值不为“online”则表示数据同步异常。 若数据同步异常，执行命令检查数据库备机和主机之间的网络连通性，数据库主机IP参考 1 获取。 ping 数据库主机IP 无法ping通，请联系网络部门协助解决，网络问题解决成功后执行 7 进行重建。 若可以ping通，执行 7 进行重建解决。 执行以下重建命令，进行备机的全量重建。 gs_ctl build -b full -M standby 执行以下重建命令，进行数据库备机增量重建。 gs_ctl build -b incremental -M standby 再次执行 5 ，判断是否成功建立复制同步关系。 父主题： 管理面应急案例 版权所有 © 华为技术有限公司
