数据库主备倒换 数据库主备倒换 注意事项 switchover为维护操作，确保集群状态正常，所有业务结束，并使用pgxc_get_senders_catchup_time()视图查询无主备追赶后，再进行switchover操作。 命令格式 cm_ctl switchover [-z AVAILABILITY_ZONE] | [-n NODEID -D DATADIR [-q] | [-f]] | [-a [-q]] | [-A] [-t SECS] 参数说明 -z AVAILABILITY_ZONE：重置节点状态到指定AZ，AVAILABILITY_ZONE为AZ名称，可通过cm_ctl query -Cvz ALL命令查询。 -n NODEID：重置节点状态到指定节点的实例，NODEID为节点名称，可通过cm_ctl query -Cv命令查询。-n参数需要和-D参数一起使用。 -D DATADIR：重置节点状态到指定节点的实例，DATADIR为指定实例数据目录，可通过cm_ctl query -Cvd命令查询。-D参数需要和-n参数一起使用。 -f：指定进行-f类型switchover。-f参数需要和-n以及-D参数一起使用。 -a：重置节点状态为初始配置状态。 -A：将所有GTM、DN实例统一从主切换到备。 -t SECS：指定超时时间。超时后，会退出并报错。不设置该参数时，cm_ctl switchover命令默认超时时间为120s。 常用命令 指定AZ的数据库主备倒换： cm_ctl switchover -z AZ1 指定节点的数据库主备倒换： cm_ctl switchover -n 1 -D /data1/omm/cluster/data/datanode1p 将所有GTM、DN实例统一从主切换到备： cm_ctl switchover -A 重置节点状态为初始配置状态： cm_ctl switchover -a 父主题： cm_ctl工具使用指南 版权所有 © 华为技术有限公司
