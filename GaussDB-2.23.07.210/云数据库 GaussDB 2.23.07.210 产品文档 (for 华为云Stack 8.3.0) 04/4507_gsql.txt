使用gsql操作内存解密逃生通道 使用gsql操作内存解密逃生通道 全密态数据库要开启内存解密逃生通道，gsql需要开启-C3或--enable-client-encryption=3。 使用gsql操作内存解密逃生通道全密态数据库需要完成密钥配置相关操作，请参见 使用gsql操作密态数据库 。 操作步骤 执行以下命令打开密态开关，连接密态数据库。 gsql -p PORT -d gaussdb -h host -U user -W password -r -C3 通过元命令设置访问主密钥的参数 注意：从keyType字符串开始，不要添加换行，不要添加空格，否则gsql工具无法识别完整参数 华为云支持两种认证方式，两种认证方式的参数个数与参数类型不同，选择其中一种方式即可 认证方式一 aksk认证 gaussdb=# \key_info keyType=hcs_kms,kmsProjectId={资源ID},ak={AK},sk={SK} 参数获取： 生成主密钥阶段介绍了如何获取相关参数：项目ID、AK、SK 示例： \key_info keyType=hcs_kms,kmsProjectId=0b59929e8100268a2f22c01429802728,ak=XMAUMJY******DFWLQW,sk=ga6rO8lx1Q4uB*********2gf80muIzUX 认证方式二 账号密码认证 gaussdb=# \key_info keyType=hcs_kms,iamUrl={ IAM服务器地址 },iamUser={用户名},iamPassword={用户密码},iamDomain={所属租户},kmsProject={资源空间名称} 参数获取： 生成主密钥阶段 介绍了如何获取上述提到的5个参数。 示例：\key_info keyType=hcs_kms,iamUrl=https://iam.xxx.com/v3/auth/tokens,iamUser=test_user,iamPassword=*********,iamDomain=test_account,kmsProject=xxx 定义主密钥 在生成主密钥阶段，密钥服务已生成并存储主密钥，执行本语法只是将主密钥的相关信息存储在数据库中，方便以后访问。该语法详细格式参考：《开发者指南》中“SQL参考 > SQL语法 > CREATE CLIENT MASTER KEY”章节。 gaussdb=# CREATE CLIENT MASTER KEY cmk1 WITH ( KEY_STORE = hcs_kms , KEY_PATH = '{KMS服务器地址}/{密钥ID}', ALGORITHM = AES_256); CREATE CLIENT MASTER KEY 参数获取： 生成主密钥阶段 介绍了如何获取如下参数：KMS服务器地址、密钥ID KEY_PATH示例：https://kms-scc-apig.sa-fb-1.console-arm2.com/v1.0/0b59929e8100268a2f22c01429802728/kms/9a262917-8b31-41af-a1e0-a53235f32de9 定义列密钥 列密钥由上一步定义的主密钥加密。详细语法参考：《开发者指南》中“SQL参考 > SQL语法 > CREATE COLUMN ENCRYPTION KEY”章节 gaussdb=# CREATE COLUMN ENCRYPTION KEY cek1 WITH VALUES (CLIENT_MASTER_KEY = cmk1, ALGORITHM  = AES_256_GCM); 创建加密表并向加密表插入数据。 1
