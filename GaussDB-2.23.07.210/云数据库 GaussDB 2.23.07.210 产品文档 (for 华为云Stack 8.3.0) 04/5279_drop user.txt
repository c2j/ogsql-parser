ERROR:  role "test1" cannot be dropped because some objects depend on it 原因分析 在这种场景下，可以通过查看系统表找到该用户被其他用户赋予权限的对象，以其他用户登录数据库手动删除其赋予该用户的权限对象，再执行drop user。 处理方法 参考 登录实例节点 ，登录数据库任意一个节点。 执行如下命令，导入环境变量。 source /home/Ruby/gauss_env_file 执行如下指令，连接数据库。 gsql -d postgres -U root -p 8000 -h 10 .0.0.0 -W pwd postgres为需要连接的数据库名称，10.0.0.0为数据库所在的服务器IP地址，root为登录数据库的用户名，8000为数据库端口号，pwd为root用户的密码。 执行如下语句，查看删除失败的用户被哪些权限对象所依赖。 以用户“test1”为例。如下示例表示存在两个权限对象依赖。 gaussdb=#  select d.datname,s.classid,s.objid from pg_roles r join pg_shdepend s on r.oid=s.refobjid join pg_database d on s.dbid=d.oid where rolname=' test1 ';
