节点替换 节点替换 故障现象 GaussDB 是由多台主机组成的，当集群中主机发生故障时，为了使 GaussDB 快速地恢复正常，运维人员可以使用节点替换功能将发生故障的节点替换为正常节点。 以混合部署（4副本）为例，集群部署在多个AZ(数据中心)，如3AZ部署、同城双活等多AZ部署场景，节点上的进程损坏且无法修复或节点损坏。如下所示节点损坏，便可以使用节点替换功能替换故障节点。 图1 分布式集群混合部署示意 图2 主备版示意图 业务影响 单个节点损坏，如果节点上有主DN/GTM/CMS组件，将会发生主备切换，业务有10S的闪断；如果该节点上的组件全是备机，则对业务没有影响，但集群可靠性会降低。节点替换时，被替换的节点上不允许存在主DN，否则节点替换会失败。 注意事项 替换操作需要确保集群处于启动状态，且没有加锁。 节点替换期间，会有分钟级的加锁操作，影响实例DDL操作。 登录状态正常且包含CN节点。 执行如下命令，连接数据库。 gsql -d postgres -h 10.0.0.0 -U root -p 8000 postgres为需要连接的数据库名称，10.0.0.0分布式为CN的IP地址，主备版为主DN的IP地址，root为登录数据库的用户名，8000为CN或主DN的端口号。 执行如下命令，查看集群是否加锁。 select count(1) from pg_stat_activity where instr( query, 'pgxc_lock_for_backup()') > 0 or instr( query, 'pg_advisory_lock(65535,65535)') > 0; 如果值小于等于1，说明锁不存在，可直接执行节点替换。 大于1表示锁存在，需要与用户确认优先级，根据业务情况选择等待用户DDL结束还是手动解锁。 解锁用户锁的命令如下： gsql -d database -U root -p port -W XXX -X -f $GAUSSHOME/bin/script/gspylib/etc/sql/unlock_cluster.sql xxx需要替换为数据库密码。 替换操作需要在一个正常主机上执行。 实例磁盘满状态下不允许进行节点替换。 替换的主机上不能含有主DN。 集群内下列组件中分别至少存在一个正常运行的：CM Server、CM Agent、CN（仅分布式混合部署）。 正常的ETCD个数必须大于ETCD总个数的一半。 如果集群中部署有GTM，需保证至少存在一个正常运行的GTM实例（仅分布式集群混合部署）。 修复CN过程中，在CN状态未变为Normal前，不能连接该CN执行业务（仅分布式集群混合部署）。 节点替换完成后，需要登录告警平台查看是否有该节点的节点状态异常告警，如果有告警需要手动清除。 ETCD异常时，替换需要等待一段时间才可执行。 预期处理时长 与节点数据量有关。 日志收集要求 参考 下载租户节点日志 收集各模块日志路径。 Agent： /home/Ruby/log/asyncJobResult.dat /home/Ruby/log/agent.log GaussDB Kernel： /var/lib/log/Ruby 处理思路 在DBS运维管理平台上通过节点替换功能，用新节点替换故障的节点。 故障定位 某节点上的CN/DN/GTM/ETCD/CMS发生故障，或整个节点故障，收到故障告警。 在DBS运维管理平台 实例管理的实例列表中单击实例名称，在实例基本信息页查看节点状态。 图3 节点状态 故障恢复 检查底层资源。 登录Service OM平台。 BMS场景下选择“服务列表 > 裸金属资源”。 在故障节点所在AZ，根据规格查看是否有可部署的裸金属服务器。 ECS、MCS场景下选择“服务列表 > 计算资源”。 在故障节点所在AZ“主机组”页签，查看CPU和内存是否充足（MCS需查看磁盘剩余空间是否充足）。 登录DBS运维管理平台 。 在“实例运维 > 实例管理”页签根据实例ID找到故障实例。单击实例名称。 找到故障节点，在对应节点的“操作”列，单击 “更多 >节点修复” 。 图4 节点修复 在弹出框中选择“节点替换”，单击“确定”。 确认集群均衡状态。 参考 登录实例节点 ，登录数据库节点。 执行如下命令，导入环境变量。 source /home/Ruby/gauss_env_file 执行如下指令，查询集群状态。 c m_ctl query -Civd 如图所示，balanced参数为“No”，说明集群主备分布不均衡。需要继续执行下列步骤，使主备分布均衡，如果主备分布已经均衡，则不需要执行如下步骤。 执行如下指令，使集群主备分布均衡。 cm_ctl switchover -a 该操作建议在业务压力较小时执行。 执行如下指令，查询主备分布是否均衡。 cm_ctl query -Civd 如图所示，balanced参数为“Yes”，说明集群主备分布均衡。 恢复确认 在DBS运维管理平台查看节点状态是否恢复正常。 父主题： 租户面应急案例 版权所有 © 华为技术有限公司
