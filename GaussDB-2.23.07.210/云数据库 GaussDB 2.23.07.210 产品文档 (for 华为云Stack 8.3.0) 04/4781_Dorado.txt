Dorado存储卷映射配置 Dorado存储卷映射配置 进入Service OM界面。 选择“服务列表 > 存储资源 > 磁盘类型”，获取新扩容的Dorado AZ（第4章节中新扩容的AZ）对应的磁盘类型名称，继续执行下列操作。 磁盘类型参考步骤 2 。 在GaussCore节点数据库中执行如下语句，打开磁盘类型映射开关。 UPDATE DBS_SYSTEM_PARAMETER  SET VALUE = 'true' WHERE NAME LIKE '%VOLUME_TYPE_CORRESPONDING_SWITCH%'; 一个AZ中存在单个Dorado磁盘类型 执行如下SQL，查询当前GaussCore库中已有AZ的磁盘类型名称： SELECT * FROM DBS_SYSTEM_PARAMETER WHERE NAME LIKE '%VOLUME_TYPE_AZ_CORRESPONDING_MAP%'; 例如，本次新增AZ为："az16.dc7"，其对应的磁盘类型为： "az16.dc7":{"DORADO":"business_dorado_02" }。 执行UPDATE语句，更新新增AZ的Dorado磁盘类型。 UPDATE DBS_SYSTEM_PARAMETER SET VALUE = $map WHERE NAME LIKE '%VOLUME_TYPE_AZ_CORRESPONDING_MAP%'; 执行UPDATE命令前要提前查询VOLUME_TYPE_AZ_CORRESPONDING_MAP，新增AZ的Dorado磁盘类型需要追加到查询的$map中，不能覆盖原有数据。 $map 配置如下 ： core中查询数据为： '{"az1":{"SSD":"SSD_AZ1","SATA":"SATA_AZ1"},"az2":{"SSD":"SSD_AZ2","SATA":"SATA_AZ2"},"az3":{"SSD":"SSD_AZ2","SATA":"SATA_AZ2"},"az15.dc7":{"DORADO":"business_dorado_01"}}' 本次需要新增"az16.dc7"，则需要修改如下： '{"az1":{"SSD":"SSD_AZ1","SATA":"SATA_AZ1"},"az2":{"SSD":"SSD_AZ2","SATA":"SATA_AZ2"},"az3":{"SSD":"SSD_AZ2","SATA":"SATA_AZ2"},"az15.dc7":{"DORADO":"business_dorado_01"}, "az16.dc7":{"DORADO":"business_dorado_02" }}' 其中“business_dorado_02”为步骤 2 中获取的"az16.dc7"对应的磁盘类型名称。 一个AZ存在多个Dorado磁盘类型 例如，新增AZ为"az16.dc7"，其对应的磁盘类型有："disk_fc_multi_pool_001,disk_fc_multi_pool_002,business_dorado_02"。 执行UPDATE语句，更新新增AZ的Dorado磁盘类型 UPDATE DBS_SYSTEM_PARAMETER SET VALUE = $map WHERE NAME LIKE '%VOLUME_TYPE_AZ_CORRESPONDING_MAP%'; 执行UPDATE命令前要提前查询VOLUME_TYPE_AZ_CORRESPONDING_MAP，新增AZ的Dorado磁盘类型需要追加到查询的$map中，不能覆盖原有数据。 $map 配置如下 ： core中查询数据为： '{"az1":{"SSD":"SSD_AZ1","SATA":"SATA_AZ1"},"az2":{"SSD":"SSD_AZ2","SATA":"SATA_AZ2"},"az3":{"SSD":"SSD_AZ2","SATA":"SATA_AZ2"},"az15.dc7":{"DORADO":"business_dorado_01"}}' 本次需要新增"az16.dc7"，则需要修改如下： '{"az1":{"SSD":"SSD_AZ1","SATA":"SATA_AZ1"},"az2":{"SSD":"SSD_AZ2","SATA":"SATA_AZ2"},"az3":{"SSD":"SSD_AZ2","SATA":"SATA_AZ2"},"az15.dc7":{"DORADO":"business_dorado_01"}," az16.dc7":{"DORADO":"disk_fc_multi_pool_001,disk_fc_multi_pool_002,business_dorado_02"}} ' 配置发放dorado多池实例的volumeType： 登录DBS运维管理平台，选择“实例运维->配置管理 -> 系统参数管理”，在右侧下拉框选择“DBS_STSTEM_PARAMETER”表 登录DBS运维管理平台，选择“实例运维->配置管理 -> 系统参数管理”，在右侧下拉框选择“DBS_STSTEM_PARAMETER ”表 搜索框输入参数名“DORADO_VOLUME_ANTIAFFINITY”的value值，true表示支持多池dorado实例创建，false表示不支持dorado多池实例的创建。 当前需要创建至少两个磁盘类型，一个磁盘类型配置一个存储池，用于发放GaussDB数据库实例的共享卷，另外一个磁盘类型至少配置三个存储池（每个存储池都分别对应不用的存储设备），用户发放GaussDB数据库的数据卷实现多池。 登录DBS运维管理平台，选择“实例运维->配置管理 -> 系统参数管理”，在右侧下拉框选择“DBS_STSTEM_PARAMETER ”表 搜索框输入参数名“DORADO_POOL_PREFIX”输入共享卷的磁盘类型前缀信息，用来识别当前哪些池是Dorado多池。 修改系统参数SHARE_DORADO_VOLUME。 登录DBS运维管理平台，选择“实例运维->配置管理 -> 系统参数管理”，在右侧下拉框选择“DBS_STSTEM_PARAMETER ”表。 搜索框输入参数名“SHARE_DORADO_VOLUME”，填写参数值，用来指定新增AZ中创建实例使用的Dorado共享卷对应的存储池。 SHARE_DORADO_VOLUME参数值配置示例如下：{"az16.dc7":"business_dorado_02","az18.dc7":"business_dorado_03"} az16.dc7：新增AZ；business_dorado_02：新增AZ中共享卷使用的存储池。 父主题： 元数据库配置 版权所有 © 华为技术有限公司
