灾备实例节点故障，升主后下发节点修复任务失败 灾备实例节点故障，升主后下发节点修复任务失败 灾备升主流程中由于服务器宕机，网络中断等原因导致节点处于故障状态脱离灾备集群，没有参与灾备集群升主，对故障节点下发节点修复任务失败。本章介绍修复方案 前提条件 灾备实例存在节点异常，灾备升主操作流程已完成 操作步骤 故障节点经过硬件维修、上电等操作，重新加入集群。 参考 登录实例节点 ，接入任意节点。 修改cm_service合cm_agent中关于灾备实例模式参数，切回主结群配置（如果是沙箱模式需要/usr/sbin/chroot/var/chroot进入沙箱，然后加载环境变量）。 gs_guc set -Z cmserver -N all -I all -c "backup_open = 0" gs_guc set -Z cmagent -N all -I all -c "agent_backup_open=0" gs_guc set -Z cmagent -N all -I all -c "disaster_recovery_type= 0" 参考 登录实例节点 ，登录故障节点，查询cm_server和cm_agent的进程ID，使用kill -9命令杀掉进程，然后进程会被om_monitor重启，完成cm_server和cm_agent参数修改的生效。 ps -ef | grep cm_agent ; ps -ef | grep cm_server 使用命令获取故障节点的nodeId和未参和升主实例的coordinator和datanode路径。 cm_ctl query -Cvd 使用命令停掉该节点上未参与升主的实例。 cm_ctl stop -n {NODEID} -D {DATADIR} 下发节点修复。 父主题： 常见问题 版权所有 © 华为技术有限公司
