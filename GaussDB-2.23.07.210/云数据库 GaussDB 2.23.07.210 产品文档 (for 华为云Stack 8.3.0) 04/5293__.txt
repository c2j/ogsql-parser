节点扩容/变更部署形态失败 节点扩容/变更部署形态失败 现象描述 节点扩容或者变更部署形态失败。在DBS运维管理平台查看任务树，任务失败在“ExpandCnSyncNodeFlowLimitDataTask”，并且报错信息中包含“reseverParseRuleData”信息。 可能原因 SQL限流关键词解析失败。 定位思路 查找并删除解析失败的SQL限流任务。 处理步骤 登录管理控制台。 查看SQL限流任务关键词中是否包含以下几种特殊字符。 字符串内容有 " 或 \ 或 {}。 字符串除了首末位，包含空格。 删除原节点包含特殊字符的限流任务。 删除新节点包含特殊字符的限流任务。 a. 登录管理控制台，点击实例进入基本信息页面，找到新增节点nodeId，然后遍历登录 b. 参考 登录实例节点 ，登录实例节点。 c. 执行如下命令，连接实例。 gsql -p 8000 -d postgres -r -U root d. 执行如下语句，查询当前已设定的限流规则。 select * from gs_workload_rule; e. 根据查看表中记录option_val字段里除去外层的大括号是否还包含前面步骤2中 提到的几种特殊字符，如果存在则根据记录的rule_id作为删除条件，执行如下语句清理限流规则数据。 select gs_delete_workload_rule(rule_id) from gs_workload_rule where rule_id in (' rule_id'); 举例：如6.5.3-4.d中查询结果有rule_id 为“2”，“3”，“4”的记录里option_val字段除掉最外层的大括号分别是“"\"asd"”，“"asds asdasd"”，“"{sdfs}"”仍都存在特殊字符，则执行语句： select gs_delete_workload_rule(rule_id) from gs_workload_rule where rule_id in ('2','3','4'); 在节点扩容/变更部署形态任务树中重试SQL限流同步工步。 父主题： DBMind管理 版权所有 © 华为技术有限公司
