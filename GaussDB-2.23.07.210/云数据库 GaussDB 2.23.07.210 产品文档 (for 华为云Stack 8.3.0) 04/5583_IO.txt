IO满应急处理 IO满应急处理 故障现象 业务高峰期、跑批、marge、update、备份等操作引发IO过高。 影响范围 IO使用率太高会影响该台主机数据库的正常运行,应用运行缓慢。 快速恢复 参考 登录实例节点 ，登录IO满的节点。 查看IO时延等监控或者使用命令查看IO情况。 iotop查看是否是GaussDB进程占用。 若为非数据库进程占用，需要排查异常进程归属。 查询集群状态，确认各组件状态是否正常。 cm_ctl query -Cvid 若组件正常，继续 4 。 若组件状态异常，则需要联系华为技术支持。 若gaussdb进程所在磁盘IO冲高，通过gsql连接数据库。 gsql -d dbname -U user_name -p 8000 -W password -r user_name为数据库用户名，password为数据库密码,端口为步骤4查出来的DN端口（分布式为CN端口）,dbname为实际数据库名。 查看长时间运行的会话情况。 集中式执行： select pid,sessionid, substring(query,0,100), state,usename,now()-xact_start as elp from pg_stat_activity where  usename != 'rdsAdmin' and state != 'idle' order by elp desc; 分布式执行： select pid,sessionid, substring(query,0,100), state,usename,now()-xact_start as elp from pgxc_stat_activity where  usename != 'rdsAdmin' and state != 'idle' order by elp desc; 查看是否有长时间未结束的SQL，例如elp大于5min，记录按照执行时间排序的TOP 10 SQL访问的表。 对步骤5记录的表执行analyze收集统计信息。 analyze table_name table_name为 步骤5 收集的表 若IO使用率未见明显下降，对整库执行analyze。 analyze 若IO使用率未见明显下降，kill慢SQL。 集中式执行： select pg_terminate_session(pid,sessionid) from pg_stat_activity where usename != 'rdsAdmin' and state != 'idle'; 分布式执行： execute direct on ALL 'select pg_terminate_session(pid,sessionid) from pg_stat_activity where usename != ''rdsAdmin'' and state != ''idle'';'; 此步骤可执行2~5次，同时查询CPU使用率是否降低。 停止正在执行的SQL语句为高危操作，执行前需联系客户及华为技术确认。 若第8步执行后IO使用率未明显下降，尝试进行主备切换。 cm_ctl switchover -n nodeid -D dataDir nodeid为待切换为主的节点id，dataDir为数据目录。 高危操作，建议联系华为工程师后再执行。 若主备切换后问题依旧未解决，尝试联系客户停止部分业务。 高危操作，客户与华为侧相关人员达成一致意向后由客户进行操作。 恢复确认 查看IO利用率是否下降。 iostat -xm 1 100 IO使用率恢复至业务高峰以下。 父主题： 资源类故障应急案例 版权所有 © 华为技术有限公司
