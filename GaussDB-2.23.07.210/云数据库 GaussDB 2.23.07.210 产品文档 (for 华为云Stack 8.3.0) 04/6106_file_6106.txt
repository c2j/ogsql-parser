操作系统参数配置建议 操作系统参数配置建议 数据库实例安装完成后，在SATA/SAS SSD下磁盘调度方式建议使用deadline调度方式，Nvme建议使用none调度方式。 该参数对磁盘性能影响较大，设置不当会影响数据的读写性能，设置方法参考如下步骤。 操作步骤 参考 登录实例节点 ，登录实例节点。 切到root用户。 exit 查看当前磁盘的调度方式： cat /sys/block/ $DEVICE-NAME /queue/scheduler $DEVICE-NAME代表具体的磁盘设备，如sdb。 noop deadline [cfq] [ ]中的值即为当前使用的磁盘IO调度模式。 使用echo方式临时修改磁盘的调度方式。 例如将sdb磁盘修改为deadline： echo deadline > /sys/block/sdb/queue/scheduler 持久化修改磁盘的调度方式。 查看磁盘调度算法 cat /sys/block/sdb/queue/scheduler mq-deadline kyber [bfq] none 编写udev文件 vim /etc/udev/rules.d/io_scheduler.rules ACTION=="add|change",KERNEL=="sdb",ATTR{queue/scheduler}="mq-deadline" 修改磁盘udev规则 udevadm trigger --name-match=sdb 查看磁盘调度算法 cat /sys/block/sdb/queue/scheduler [mq-deadline] kyber  bfq none 父主题： 麒麟镜像制作指导 版权所有 © 华为技术有限公司
