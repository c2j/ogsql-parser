概述 概述 本文档主要对 GaussDB 日志进行介绍，同时提供了查看日志的方法。 日志介绍 日志记录了系统的运行情况和功能流程的处理情况。日志不仅是维护工程师查看系统状态、进行问题定位的重要工具和手段，也是用户收集故障信息的基础。 日志的功能包括但不限于： 记录系统在运行中的状态、动作信息。 检测并记录内部错误，记录错误发生条件、环境、轨迹等。 GaussDB 日志可分为管理日志和租户日志。 管理日志：用于记录管理面虚机的操作日志。 租户日志：用于收集云服务运行过程中的日志。 文档将通过介绍各运行日志的日志含义、日志格式、日志参数，日志示例等，帮助维护工程师解读日志，快速定位问题。 日志含义：描述某日志记录的信息或该日志的具体用途。 日志命名：描述某日志的具体命名规则。 日志格式：描述某日志的内容构成，包含哪些信息要素 。 日志参数：解释某日志内容中的各项信息要素的含义 。 日志参数中一般都会包含日志级别信息，日志级别和其对应的含义如 表1 所示。 表1 日志级别 级别名称 级别含义 DEBUG 记录系统的工作过程，包括软件工作过程中各操作的日志信息以及相应的参数。 INFO 记录系统运行的工作流程以及工作流程中涉及的关键步骤。 WARN 记录系统运行过程中，系统可能出现的问题或异常。 ERROR 记录系统运行过程中发生的错误。 日志示例：提供某日志的典型样例 。
