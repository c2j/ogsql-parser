生成证书文件 生成证书文件 本章指导用户在备份服务器生成证书文件。 注意事项 在以下生成证书文件过程中，需要自行设定根私钥、服务端私钥、客户端私钥的密码，密码的设定要求如下： 长度在8-15位之间。 至少包含一个大写字母、一个小写字母、一个数字和一个特殊字符。 特殊字符的输入不支持 '（单引号）、"（双引号）、 ` (反引号)以及空格字符。 请按照上述要求设定密码，不符合上述要求的密码可能会导致证书配置失败或SSL认证失败。 操作步骤 搭建CA环境 。 以 myuser 用户登录备份服务器，执行如下命令，进入/home/myuser/data目录， cd ~ /data 执行如下命令，在/home/myuser/data文件夹下创建sslcert目录并进入该目录： mkdir sslcert && cd sslcert 执行如下命令，将openssl相关配置文件拷贝到当前目录： cp ../backUpAgent/bin/openssl ../backUpAgent/bin/openssl.cnf . 在sslcert文件夹下通过如下命令建立CA环境： mkdir ./demoCA ./demoCA/newcerts ./demoCA/private
chmod 777 ./demoCA/private
echo '01'> ./demoCA/serial
touch ./demoCA/index.txt 生成根私钥、根证书 。 执行如下命令，生成根私钥，执行命令时在提示框设定根私钥的密码，以下以<root_password>指代该密码，请妥善记住该密码： ./openssl genrsa -aes256 -out demoCA/private/cakey.pem 2048 执行如下命令，生成根证书请求，执行命令时在提示框输入根私钥的密码 <root_password>： ./openssl req -config ./openssl.cnf -new -key demoCA/private/cakey.pem -sigopt rsa_padding_mode:pss -out demoCA/careq.pem -subj '/C=CN/ST=Beijing/L=Beijing/O=huawei/OU=gauss/CN=root' 执行如下命令，生成根证书，执行命令时在提示框输入根私钥的密码 <root_password>： ./openssl ca -config ./openssl.cnf -batch -out demoCA/cacert.pem -keyfile demoCA/private/cakey.pem -extensions v3_ca -selfsign -infiles demoCA/careq.pem 修改 demoCA/index.txt.attr 文件，将 unique_subject 属性修改为 no，并使用 :wq 退出保存： vim demoCA/index.txt.attr 生成服务端私钥、服务端证书 。 执行如下命令，生成服务端私钥，执行命令时在提示框设定服务端私钥的密码，以下以<server_password>指代该密码，请妥善记住该密码： ./openssl genrsa -aes256 -out server.key 2048 执行如下命令，生成服务端证书请求文件，执行命令时在提示框输入服务端私钥的密码<server_password>： ./openssl req -config ./openssl.cnf -new -sigopt rsa_padding_mode:pss -key server.key -out server.req -subj '/C=CN/ST=Beijing/L=Beijing/O=huawei/OU=gauss/CN=server' 执行如下命令，通过根私钥、根证书颁发生成服务端证书，执行命令时在提示框输入根私钥的密码 <root_password>： ./openssl ca -config ./openssl.cnf -extensions server -batch -in server.req -out server.crt -subj '/C=CN/ST=Beijing/L=Beijing/O=huawei/OU=gauss/CN=server' 生成服务端证书时，请确认commonName字段（即上述 -subj 后字符串中的CN字段）设置为 server。 生成客户端私钥、客户端证书 。 执行如下命令，生成客户端私钥，执行命令时在提示框设定客户端私钥的密码，以下以<client_password>指代该密码，请妥善记住该密码： ./openssl genrsa -aes256 -out client.key 2048 执行如下命令，生成客户端证书请求文件，执行命令时在提示框输入客户端私钥的密码<client_password>： ./openssl req -config ./openssl.cnf -new -sigopt rsa_padding_mode:pss -key client.key -out client.req -subj '/C=CN/ST=Beijing/L=Beijing/O=huawei/OU=gauss/CN=client' 执行如下命令，通过根私钥、根证书颁发生成客户端证书，执行命令时在提示框输入根私钥的密码 <root_password>： ./openssl ca -config ./openssl.cnf -extensions client -batch -in client.req -out client.crt -subj '/C=CN/ST=Beijing/L=Beijing/O=huawei/OU=gauss/CN=client' 生成客户端证书时，请确认commonName字段（即上述 -subj 后字符串中的CN字段）设置为 client。 指定文件权限并清理无用文件 。 执行如下命令，指定证书文件权限为600，并删除证书请求文件： cp demoCA/cacert.pem ./
