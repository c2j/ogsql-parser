备集群failover之后的未参加升主的节点修复 备集群failover之后的未参加升主的节点修复 因服务器宕机，网络中断等原因导致节点脱离灾备数据库实例，没有参与灾备数据库实例升主。在服务器恢复之后，该节点会一直处于异常状态，需要手动进行修复。 注意事项 在修复前，该节点会一直处于standby need repair状态。如下图中6002的节点状态 操作步骤 对未参与升主的节点，可以按照如下流程进行修复： 登录到集群上任意节点，修改CMS和CMA中关于数据库实例备集群模式的参数，切回主数据库实例配置。 gs_guc set -Z cmserver -N all -I all -c "backup_open = 0"
gs_guc set -Z cmagent -N all -I all -c "agent_backup_open=0" 接入故障节点，查询CMS和CMA的进程ID，使用kill -9命令杀掉进程，然后进程会被om_monitor重启，完成CMS和CMA参数修改的生效。 使用cm_ctl stop -n NODEID -D DATADIR停掉该节点上未参与升主的DN实例。 使用DN实例进行节点修复（修复步骤可参见《GaussDB应急预案》4.2.3章节）。 父主题： Dorado容灾故障手册 版权所有 © 华为技术有限公司
