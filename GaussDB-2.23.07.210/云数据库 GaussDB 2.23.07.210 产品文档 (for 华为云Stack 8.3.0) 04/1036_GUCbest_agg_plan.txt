案例：调整GUC参数best_agg_plan 案例：调整GUC参数best_agg_plan 现象描述 agg_ t1的表定义为： 1 create table agg_t1 ( a int , b int , c int ) distribute by hash ( a ); 假设agg下层算子所输出结果集的分布列为setA，agg操作的group by列为setB，则在Stream框架下，Agg操作可以分为两个场景。 setA是setB的一个子集。 对于这种场景，直接对下层结果集进行汇聚的结果就是正确的汇聚结果，上层算子直接使用即可。如下图所示： 1
