主CN剔除， 备无法连接，RPO增高处理 主CN剔除， 备无法连接，RPO增高处理 搭建容灾关系后，由于主集群具有容灾关系的CN状态异常，或者由于网络异常，灾备集群对应CN无法连接，RPO持续增长。本章介绍恢复RPO的处理流程。 前提条件 主备集群的容灾关系正常。 操作步骤 参考 登录实例节点 ，登录主实例节点。 执行如下指令，检查主集群状态。 cm_ctl query -Cvid 参考 登录实例节点 ，登录灾备集群节点。 执行如下指令，检查灾备集群状态。 cm_ctl query -Cvid 如图所示，CN状态为“Disconnected”的即为需要停止的CN节点。 执行如下命令，停止灾备实例相关CN节点。 cm_ctl stop -n NODEID -D DATADIR NODEID需要替换为状态为“disconnected”的CN ID，DATADIR需要替换为CN节点的目录。 停止灾备实例CN后RPO可恢复。 等主实例具有容灾关系的CN修复后，可执行如下命令重启灾备实例对应的CN。 cm_ctl start -n NODEID -D DATADIR NODEID需要替换为已经停止的CN ID，DATADIR需要替换为CN节点的目录。 父主题： 容灾管理 版权所有 © 华为技术有限公司
