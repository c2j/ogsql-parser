磁盘故障 磁盘故障 数据库的数据盘使用的dorado存储的LUN盘，当dorado存储与机器之间的网络出现故障时，会导致磁盘不可用。网络恢复之后可能因为内存和磁盘的数据不一致，导致dorado磁盘目录无法访问。 注意事项 如果按照以下步骤无法修复，可以通过重启机器进行恢复。 操作步骤 使用root用户登录故障节点，然后执行命令。 lsof -t /var/chroot/var/lib/engine/data1 |xargs kill -9 && umount -f /var/chroot/var/lib/engine/data1 lsof -t /var/chroot/usr/local/etcd |xargs kill -9 && umount -f /var/chroot/usr/local/etcd lsof -t /var/chroot/var/lib/log/backup |xargs kill -9 && umount -f /var/chroot/var/lib/log/backup lsof -t /var/chroot/var/lib/log |xargs kill -9 && umount -f /var/chroot/var/lib/log 对对应的磁盘执行修复命令。 fsc k.ext4 /dev/mapper/gaussdbvg1-mydata1 fsc k.ext4 /dev/mapper/etcdvg-etcddata fsc k.ext4 /dev/mapper/backupvg-backupdata fsc k.ext4 /dev/mapper/logvg-logdata 重新挂载磁盘。 mount -a 确认磁盘已经修复，执行如下命令，可以查看到该目录下的文件。 ls /var/chroot/var/lib/engine/data1 ls /var/chroot/usr/local/etcd ls /var/chroot/var/lib/log/backup ls /var/chroot/var/lib/log 父主题： Dorado容灾故障手册 版权所有 © 华为技术有限公司
