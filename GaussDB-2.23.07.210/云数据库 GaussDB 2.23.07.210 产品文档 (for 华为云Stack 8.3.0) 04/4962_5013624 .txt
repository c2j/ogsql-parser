5013624 同步实例参数失败 5013624 同步实例参数失败 告警解释 在DBS运维管理平台，用户下发参数同步后，若在同类型组件下不同节点中同一个guc参数不相同，则触发告警。 告警属性 告警ID 告警级别 是否可自动清除 5013624 次要 否 告警参数 参数名称 参数含义 名称 同步实例参数失败。 类型 操作告警。 发生时间 告警产生时间。 定位信息 实例ID：产生告警的实例ID 系统=com/huawei/hwclouds/dbs/common/service/common,模块=comonService,实例ID=${instanceId} 对系统的影响 无。 可能原因 用户在节点上修改guc参数未同步到其他节点。 内核默认不同节点参数不同。 处理步骤 使用浏览器，通过地址“https:// ManageOne运维面主页的访问地址 :31943”，登录ManageOne运维面，或通过地址“https:// ManageOne主门户的访问地址 ”，登录ManageOne主门户，选择“运维中心（OC）”，进入ManageOne运维面。 密码方式：输入帐号和密码。 默认帐号：bss_admin 如果是升级场景，默认帐号为admin。 默认密码：参见 《华为云Stack 8.3.0 帐户一览表》 的“A类（Portal）”页签中，“ManageOne运维面”帐户对应的默认密码。 USB Key方式：插入已预置用户证书的USB Key，选择设备和用户证书，并输入PIN码 在“集中监控 > 告警管理 > 当前告警”中查看告警信息，获取告警中的实例ID（instance_id）和报错的参数名称“paramnames”。 参考 登录实例节点 ，根据报错的实例ID，登录实例节点。 通过gsql连接到数据库。 gsql -d postgres -U root -p 8000 -h 10.0.0.0 -W pwd postgres为需要连接的数据库名称，10.0.0.0为数据库所在的服务器IP地址，root为登录数据库的用户名，8000为数据库端口号，pwd为root用户的密码。分布式需要连接到含有CN组件的节点，主备版需要连接到含有主DN的节点。 执行如下语句，查看实例参数值。 分布式： EXECUTE DIRECT ON ALL 'SELECT pgxc_node_str() AS node_name, name, setting FROM pg_settings WHERE name IN ('' failed_login_attempts '')'; 主备版： SELECT pgxc_node_str() AS node_name, name, setting FROM pg_settings WHERE node_name LIKE 'dn%' AND name IN (' audit_system_object '); failed_login_attempts请替换为失败的参数值。 如果参数取值和预定结果不一致，请在DBS运维平台修改参数值。 参考信息 无。 父主题： 告警处理 版权所有 © 华为技术有限公司
