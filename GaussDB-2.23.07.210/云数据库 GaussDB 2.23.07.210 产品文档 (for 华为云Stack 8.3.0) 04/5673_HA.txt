查看实例的HA状态和历史操作 查看实例的HA状态和历史操作 DBS运维管理平台 提供查询HA的历史操作记录，包含实例的资源修复动作。还可以查看当前实例的HA信息，包括实例状态。 操作步骤 登录DBS运维管理平台 。 在 “实例运维 > 实例管理” 的 “实例列表” 页签查找实例，单击目标实例名称。 在新页面中，单击“HA信息”，查看实例的HA详细信息。 监控策略：监控策略无需设置。系统采用的默认监控方式为：实例异常时，优先进行自动恢复（repair）；自动修复如果不成功，会启用主备倒换（failover）；主备倒换（failover）仍然无法成功后，需人工介入，手动修复（manual）。 历史操作记录中支持的“操作类型”根据资源类型的不同，所代表的的含义也不同。 表1 操作类型 资源 操作 说明 node-db start 启动GaussDB内核进程。 GaussDB-dn repair Dorado存储实例启动共享存储重新挂载。 data-ip start 挂载数据IP。 stop 卸载数据IP。 data-vip start 挂载数据VIP。 stop 卸载数据VIP。 manage-ip start 挂载管理IP。 repair 修复数据VIP。 replica-ip start 挂载高速网卡IP。 gaussdb-failover repair 一主一备形态HA强切备为主。 data-disk start 挂载数据盘。 stop 卸载数据盘。 log-disk start 挂载日志盘。 stop 卸载日志盘。 machine-{ecsId} start 重启虚拟机。 metric start 重启节点指标功能。 kafka-client start 重启节点kafka上报功能。 node-exporter restart DBMind引擎实例重启节点系统监控。 cmd-exporter restart DBMind引擎实例重启命令行程序。 gaussdb-exporter restart DBMind引擎实例重启数据库监控。 父主题： 实例公共操作 版权所有 © 华为技术有限公司
