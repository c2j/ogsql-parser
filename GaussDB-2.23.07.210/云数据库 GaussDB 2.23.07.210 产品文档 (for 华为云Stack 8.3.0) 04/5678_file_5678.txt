节点修复和节点替换 节点修复和节点替换 数据库集群由多台主机组成，当集群中主机发生故障时，为了使数据库快速地恢复正常，可以使用该功能将发生故障的节点修复或替换为正常节点。当前替换节点只针对故障节点开放，且替换操作与自身互斥。 注意事项 节点替换： 替换操作需要确保集群处于启动状态，且没有加锁。 节点替换期间，会有分钟级的加锁操作，影响实例DDL操作。 参考 登录实例节点 ，登录对应的实例节点。 登录时，在“生成登录脚本”页，分布式版需选择节点状态正常且包含CN的节点，主备版需选择节点状态正常且包含DN的节点 。 执行如下命令，连接数据库，并输入root用户密码。 gsql -d postgres -h 10.0.0.0 -U root -p 8000 postgres为需要连接的数据库名称，10.0.0.0分布式版为CN的IP地址，主备版为主DN的IP地址，root为登录数据库的用户名，8000为CN或主DN的端口号。 执行如下命令，查看集群是否加锁。 select count(1) from pg_stat_activity where instr( query, 'pgxc_lock_for_backup()') > 0 or instr( query, 'pg_advisory_lock(65535,65535)') > 0; 如果值小于等于1，说明锁不存在，可直接执行节点替换。 大于1表示锁存在，需要与用户确认优先级，根据业务情况选择等待用户DDL结束还是手动解锁。 解锁用户锁的命令如下： gsql -d database -U root -p port -W XXX -X -f $GAUSSHOME/bin/script/gspylib/etc/sql/unlock_cluster.sql database 为需要连接的数据库名称，一般为postgres；root为登录数据库的用户名； port 为CN或主DN的端口号，一般为8000；xxx需要替换为数据库密码。 节点修复： 节点修复操作需要确保集群处于启动状态，且没有加锁。 节点修复期间，会有分钟级的加锁操作，影响实例DDL操作。 登录状态正常且包含CN节点。 执行如下命令，连接数据库。 gsql -d postgres -h 10.0.0.0 -U root -p 8000 postgres为需要连接的数据库名称，10.0.0.0分布式为CN的IP地址，主备版为主DN的IP地址，root为登录数据库的用户名，8000为CN或主DN的端口号。 执行如下命令，查看集群是否加锁。 select count(1) from pg_stat_activity where instr( query, 'pgxc_lock_for_backup()') > 0 or instr( query, 'pg_advisory_lock(65535,65535)') > 0; 如果值小于等于1，说明锁不存在，可直接执行节点修复。 大于1表示锁存在，需要与用户确认优先级，根据业务情况选择等待用户DDL结束还是手动解锁。 解锁用户锁的命令如下： gsql -d database -U root -p port -W XXX -X -f $GAUSSHOME/bin/script/gspylib/etc/sql/unlock_cluster.sql xxx需要替换为数据库密码。 如果集群中部署有ETCD，则正常的ETCD个数必须大于ETCD总个数的一半，故障的ETCD才可以被修复。 实例磁盘满状态下不允许进行节点修复。 如果故障节点所在主机的安装目录下（$GAUSSHOME/bin/）的二进制文件损坏或丢失，则不能通过该章节进行修复。需要复制其他正常主机对应的二进制文件到该主机，再通过该章节修复。 主DN存活时，故障的备DN才可以被修复。 主GTM存活时，故障的备GTM才可以被修复（仅分布式集群混合部署）。 在前一次修复结束后才能再次执行修复，禁止多个节点同时执行修复。 操作步骤 登录DBS运维管理平台 。 在 “实例管理” 的 “实例列表” 页签查找实例，单击“实例名称”进入实例的基本信息页面。 选择需要修复的节点，在操作列单击“更多”，选择“节点修复”。 在弹出框中，选择修复方式“节点替换”或“节点修复”。 节点修复：系统尝试修复损坏的组件，通常用于节点上组件损坏或者组件状态异常等，节点上进程文件损坏无法修复。 节点替换：使用新的节点替换集群中的异常节点，通常用于进程文件损坏或者整个节点无法对外提供服务。 已经搭建容灾关系的集群，如果执行了节点替换，需要参考 导入容灾配置 ，刷新容灾关系。 单击“确定”，等待操作下发完成。 父主题： GaussDB实例信息 版权所有 © 华为技术有限公司
